# Тестовое задание — форма управления учетными записями

Проект реализует форму управления учетными записями на **Vue 3 + Composition API**, **TypeScript** и **Pinia** с UI-компонентами **Element Plus**.

## Возможности

- Добавление/удаление учетных записей.
- Валидация обязательных полей по потере фокуса (input) и по изменению значения (select).
- Тип записи:
  - **LDAP** — поле «Пароль» скрывается и сохраняется как `null`.
  - **Локальная** — поле «Пароль» отображается и сохраняется как введенное значение.
- Поле «Метка» (необязательное) преобразуется в массив объектов `{ text }`, разделитель `;`.
- Состояние хранится в Pinia и сохраняется в `localStorage` для восстановления после перезагрузки страницы.

## Пошаговая инструкция по запуску в VSCode

> Ниже описан базовый сценарий. Если в среде запрещен доступ к npm registry, установка зависимостей может завершиться ошибкой 403.

1. **Установите Node.js**
   - Рекомендуется версия **18+**.
   - Проверка версии в терминале:
     ```bash
     node -v
     npm -v
     ```

2. **Откройте проект в VSCode**
   - Запустите VSCode.
   - Нажмите **File → Open Folder...** и выберите папку проекта.

3. **Откройте встроенный терминал**
   - Меню **Terminal → New Terminal**.

4. **Установите зависимости**
   ```bash
   npm install
   ```

5. **Запустите приложение в режиме разработки**
   ```bash
   npm run dev
   ```

6. **Откройте приложение в браузере**
   - В терминале появится локальный адрес, обычно `http://localhost:5173`.
   - Перейдите по ссылке в браузере.

## Структура проекта

- `src/App.vue` — основной интерфейс формы и логика валидации/обновления данных.
- `src/store/accounts.ts` — Pinia store с типами, добавлением/удалением записей и сохранением в `localStorage`.
- `src/main.ts` — подключение Vue, Pinia и Element Plus.
- `src/assets/base.css` — базовые стили.

## Логика работы формы

1. **Добавление**
   - Нажмите кнопку `+` — в конец списка добавится пустая запись.

2. **Валидация и сохранение**
   - Для текстовых полей (метка, логин, пароль) — при потере фокуса.
   - Для селекта типа записи — при изменении значения.
   - При валидности запись сохраняется в Pinia и синхронизируется с `localStorage`.

3. **Метка**
   - Вводите несколько меток через `;`.
   - В store сохранится массив объектов вида:
     ```ts
     [{ text: 'Метка 1' }, { text: 'Метка 2' }]
     ```

4. **Удаление**
   - Нажмите кнопку удаления в строке записи.

## Скрипты

- `npm run dev` — запуск в режиме разработки.
- `npm run build` — сборка проекта.
- `npm run preview` — предпросмотр собранной версии.
